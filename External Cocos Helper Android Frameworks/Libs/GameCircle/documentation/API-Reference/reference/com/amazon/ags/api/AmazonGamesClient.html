Class Overview


This class is still under development and the API is subject to change or removal at any time without notice.
The API may change on your device if it is flashed with a new build or receives an OTA update.

This is the single entry point to access the Amazon Games Client. Upon initialization, the AmazonGamesClient will begin an attempt to authorize on a background thread. Be sure to wait for the AmazonGamesCallback before using the client.

public static synchronized void initialize (Activity activity, AmazonGamesCallback callback, EnumSet<AmazonGamesFeature> features)

Initializes the AmazonGamesClient for the given activity. It is important to always call this function in your current activity's onResume() and to ensure it is executed on the UI thread.

Parameters
activity the active activity at the time of calling initialize(). Must not be null.
callback a callback to notify whether the AmazonGamesClient is ready. Must not be null.
features The features that are supported by the game. This will be displayed to the user.

public static synchronized void release ()

Releases resources and operations which only need to be running during an active game session. This should be called when the game leaves the foreground and should be paired with a call to initialize() once your game resumes in the foreground (ideally, this should be called in your activity's onPause() to compliment the calling of initialize() in your activity's onResume()).

public static synchronized void shutdown ()

Fully releases all resources and operations associated with an initialized AmazonGamesClient. Call this method when you are finished interacting with the GameCircle SDK (typically, when your game is exiting). Any subsequent interactions with the SDK will require a new call to initialize(). shutdown() should always be called when you are ready to shut down your application.
